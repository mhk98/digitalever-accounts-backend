Vendor Handover Note

আমাদের attendance system backend already ready আছে। Biometric machine integration করার জন্য নিচের বিষয়গুলো follow করতে হবে।

Required Device Capability
- machine অবশ্যই ADMS / Push support করবে
- realtime transaction send/export করতে পারতে হবে
- stable machine user ID থাকতে হবে
- employee-wise face/fingerprint enrollment support থাকতে হবে

Device Setup Requirement
- machine-এ correct date, time, timezone set করতে হবে
- LAN/internet connectivity দিতে হবে
- static/fixed IP preferable
- branch/location অনুযায়ী device identify করতে হবে

System-side Required Device Info
নিচের data final করে দিতে হবে:
- device name
- brand
- model
- serial number
- IP address
- branch
- location
- sync method
- deviceIdentifier
- apiKey

Employee Enrollment Requirement
- প্রত্যেক employee machine-এ enroll করতে হবে
- প্রত্যেক employee-এর machine user ID note করতে হবে
- same user ID later change করা যাবে না without remapping

Mapping Requirement
machine user ID-গুলো আমাদের software-এ employee-এর সাথে map করতে হবে:
- employee
- device
- machine user ID

Integration Target
Machine / middleware / WDMS থেকে আমাদের backend endpoint-এ data পাঠাতে হবে:

POST /attendance-log/realtime-ingest

Expected Payload
{
  "deviceIdentifier": "HO-ZK-01",
  "deviceUserId": "101",
  "punchTime": "2026-04-13T09:03:00+06:00",
  "punchType": "in",
  "verificationMethod": "face"
}

Required Header
x-device-key: YOUR_DEVICE_API_KEY

After First Live Setup, Must Verify
- punch data raw log হিসেবে save হচ্ছে
- employee correctly matched হচ্ছে
- duplicate punch issue হচ্ছে না
- summary তৈরি হচ্ছে
- late / overtime / holiday logic ঠিক কাজ করছে

Minimum Go-live Test
1. one employee in punch
2. one employee out punch
3. raw log saved
4. employee matched
5. attendance summary generated

Important
- machine user ID stable থাকতে হবে
- deviceIdentifier final হওয়া লাগবে
- API key correct হতে হবে
- timezone mismatch থাকা যাবে না
